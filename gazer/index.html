Gazer

This post is a simple outline of how I set up Gazer - an open source command line tool for the BI tool Looker.

Installation Outline

This isn’t a comprehensive guide to the installation and just shows the process I went through.

  • Installed rbenv to manage ruby envionrments
    • Installed required dependencies Homebrew etc.
  • Removed any references to RVM previously installed as rbenv and RVM are incompatible
    • $ RVM implode
    • Removed any reference in bash_profile
    • Restart to avoid any issues
  • Installed ruby version 2.3.3. using rbenv
  • Created a local file ‘Gazer’ * cd Desktop/Gazer && rbenv local 2.3.3

  • Checked all working using rbenv versions in $ /Desktop/Gazer

  • Create .netrc file
    • cd ~ && atom .netrc
    • Added the following to the .netrc file: 
       machine <<looker instance>>.looker.com
 login <<client ID>>
 password <<Client Secret>>
    • Restrict access to the .netrc file: $ chmod 600.netrc

Note: There is an interesting issue here when running other Python scripts using session() as the .netrc file seems to cause issues.

Examples Commands

If you fire up the terminal and fire up something like gzr look cat 297 --host <<looker instance>> then you can quickly see the JSON return for the dashboard and quickly find out what you need to know.
