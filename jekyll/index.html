A short guide to getting started with Jekyll…

Jekyll

Jekyll is a static site generator that is powered by Ruby. You lob in a bit of markdown, liquid, HTML, CSS and out comes a static site ready for deployment.

For a content-driven site like this, Jekyll is ideal. It is simple, lightweight, comes with a development server for testing and can be hosted for FREE on Github.

Installing Jekyll on Windows

  1. Download the ruby installer for Windows. Ensure to add Ruby executables to the PATH option as part of the installer.
  2. Install the Ruby DevKit to match the version of Ruby installed. Enter a path for the file without spaces like C:\RubyDevKit
  3. Open the “command prompt” with Ruby.
  4. Initialize the DevKit and bind it to the Ruby installation and install Jekyll.

Building a Jekyll Site

Adding to GitHub

Your Jekyll site can be hosted for FREE on GitHub. GitHub Pages are public web pages that are freely hosted on Github’s github.io domain. GitHub pages are actually powered by Jekyll.

When running jekyll serve you will notice that the site builds with the “/repository-name” as the baseurl. If you try and the view the site locally, it will likely fail. To test the site locally, you can serve the site with an empty string as the “baseurl” option as jekyll serve --baseurl ''. When deploying to GitHub pages the URLs will all start with the /repository-name-name and resolve correctly.

To actually sync the files with GitHub I used the GitHub Desktop app for Windows.

  1. Click the + icon and select the Add option. Select the site folder from your Jekyll site and give it the name of your “/repository-name”.
  2. “Create & Add Repository” and make the first commit of the master.
  3. Create a branch called gh-pages and commit and publish.
  4. The site is at {username}.github.io/{repo-name}.
  5. In the future any changes to the site just need to be committed and synced to update your website.
